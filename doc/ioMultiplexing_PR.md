## 测试概述

对三种 IO 多路复用模型（select/poll/epoll）进行公平的性能对比测试。

## 测试架构

所有三个服务器均采用**非阻塞 I/O 设计**以确保公平对比：

| 特性 | select | poll | epoll |
|------|--------|------|-------|
| 监听 socket | SOCK_NONBLOCK | SOCK_NONBLOCK | SOCK_NONBLOCK |
| 客户端 socket | SOCK_NONBLOCK (accept4) | SOCK_NONBLOCK (accept4) | SOCK_NONBLOCK (accept4) |
| accept 模式 | 循环直到 EAGAIN | 循环直到 EAGAIN | 循环直到 EAGAIN |
| read 模式 | 循环直到 EAGAIN | 循环直到 EAGAIN | 循环直到 EAGAIN (ET) |
| 写处理 | 部分写入循环 | 部分写入循环 | 部分写入循环 |
| TCP_NODELAY | ✅ | ✅ | ✅ |
| 事件模型 | LT | LT | ET |

## 关键发现

### 1. select：波动大，高并发不可用

- 峰值 96,993 req/s（并发 50），但 100 并发骤降至 39,872（WSL 调度抖动）
- 2000+ 超时：fd_set 遍历与拷贝成为瓶颈
- 整体方差大，表现不稳定

### 2. poll：中低并发最强，整体最均衡

| 指标 | 值 |
|-----|-----|
| 峰值 | 98,425 req/s（并发 50）|
| 平均 | 78,918 req/s |
| 稳定区间（10~500） | 71K~98K |
| 2000 并发 | 73,153 req/s（仅下降 7%）|

- 所有模型中最高的平均吞吐量
- 5000 失败（fd 数组线性扫描 O(n) 瓶颈）

### 3. epoll：高并发常青树

| 指标 | 值 |
|-----|-----|
| 峰值 | 81,235 req/s（并发 1000）|
| 平均 | 71,444 req/s |
| 稳定区间（10~5000） | 66K~81K |
| 5000 并发 | **70,472 req/s（仅下降 13%）** |

- **唯一在所有并发级别（10~5000）都能正常工作的模型**
- 峰值出现在 1000 并发，体现 O(1) 事件分发优势
- 低并发时略低于 poll（ET 模式 vs LT 模式的固有开销）
- 实现缺陷：write EAGAIN 时直接断连（应注册 EPOLLOUT 等待可写），修复后性能可进一步提升

## 结论

| 结论 | 说明 |
|------|------|
| **epoll 是高并发王者** | 唯一在 5000 并发下正常工作，峰值在 1000 达到 81K |
| **poll 是中低并发性价比之王** | <200 并发稳定 70K+，平均吞吐量全场最高 78K+ |
| **select 稳定性最差** | WSL 下波动剧烈，2000+ 直接超时 |
| **仍需改进** | write EAGAIN 时直接断连，所有模型都有此问题，修复后性能可再提升|
